## 2. 机器学习基本概念

机器学习可以理解为：

> 让模型从数据中学习规律，然后用这个规律对新数据做预测或分析。

一个典型机器学习流程是：

```text
准备数据
→ 划分训练集和测试集
→ 创建模型
→ 训练模型
→ 预测测试集
→ 评估模型效果
```

在 scikit-learn 中，最常见的写法是：

```python
model.fit(X_train, y_train)
y_pred = model.predict(X_test)
```

其中：

| 名称 | 含义 |
| --- | --- |
| `X` | 输入特征 |
| `y` | 目标标签 |
| `X_train` | 训练集特征 |
| `X_test` | 测试集特征 |
| `y_train` | 训练集真实答案 |
| `y_test` | 测试集真实答案 |
| `y_pred` | 模型预测结果 |

## 3. 监督学习和无监督学习

### 3.1 监督学习

监督学习有明确答案 `y`。

例如：

```text
输入：学习时间、出勤率、作业完成率
输出：是否通过考试
```

模型训练时会看到：

```text
特征 X + 正确答案 y
```

常见监督学习任务：

| 任务 | 说明 | 示例 |
| --- | --- | --- |
| 回归 | 预测连续数值 | 房价、销量、分数 |
| 分类 | 预测类别 | 是否患病、是否生存、数字类别 |

### 3.2 无监督学习

无监督学习没有明确答案 `y`。

模型只看到数据本身，然后自己发现结构。

常见无监督学习任务：

| 任务 | 说明 | 示例 |
| --- | --- | --- |
| 聚类 | 把相似数据分组 | 用户分群 |
| 降维 | 用更少特征表示数据 | PCA 可视化 |
| 异常检测 | 找出不正常的数据 | 异常交易、异常样本 |

## 4. 线性回归

线性回归用于回归任务。

它预测的是连续数值。

例如：

```text
根据学习时间预测考试分数
根据面积预测房价
根据历史销量预测未来销量
```

线性回归的核心思想是：

> 用一条直线或一个线性公式拟合数据规律。

简单形式：

```text
y = w*x + b
```

多个特征时：

```text
y = w1*x1 + w2*x2 + ... + b
```

常见回归评估指标：

| 指标 | 含义 |
| --- | --- |
| MAE | 平均绝对误差 |
| MSE | 均方误差 |
| RMSE | 均方根误差 |
| R2 | 模型解释能力，越接近 1 越好 |

---

## 5. 逻辑回归

逻辑回归虽然名字里有“回归”，但常用于分类任务，尤其是二分类。

例如：

```text
是否通过考试
是否患病
是否生存
是否流失
```

逻辑回归输出的是概率。

例如：

```text
预测为 1 的概率 = 0.82
```

通常判断规则是：

```text
概率 >= 0.5 → 类别 1
概率 < 0.5 → 类别 0
```

常用代码：

```python
model.predict(X_test)
model.predict_proba(X_test)
```

其中：

| 方法 | 含义 |
| --- | --- |
| `predict` | 输出最终类别 |
| `predict_proba` | 输出每个类别的概率 |

---

## 6. 决策树

决策树是一种基于规则判断的模型。

它的思想类似：

```text
如果 出勤率 <= 67.5 → 类别 0
否则 → 类别 1
```

决策树的优点：

1. 规则直观
2. 可解释性强
3. 可以处理分类和回归
4. 不太需要特征标准化

决策树的缺点：

1. 容易过拟合
2. 对数据变化比较敏感
3. 单棵树稳定性不如集成模型

---

## 7. 随机森林

随机森林是由很多棵决策树组成的模型。

它的核心思想是：

> 一棵树可能判断不稳定，多棵树一起投票会更稳。

分类任务中：

```text
多棵树投票，票数最多的类别作为结果。
```

回归任务中：

```text
多棵树预测结果取平均。
```

随机森林优点：

1. 比单棵决策树更稳定
2. 不容易过拟合
3. 适合表格数据
4. 可以查看特征重要性

特征重要性可以帮助我们理解：

```text
模型主要依赖哪些特征做判断。
```

## 8. 模型评估

分类任务不能只看模型有没有输出，还要看模型预测得准不准。

常见分类评估指标：

| 指标 | 含义 |
| --- | --- |
| accuracy | 准确率，预测正确的比例 |
| confusion matrix | 混淆矩阵 |
| precision | 预测为某类的样本里，有多少是真的 |
| recall | 真实属于某类的样本里，有多少被找出来 |
| f1-score | precision 和 recall 的综合指标 |
| support | 每个类别的样本数量 |

混淆矩阵可以看出：

```text
哪些类别预测对了
哪些类别预测错了
错成了什么类别
```

二分类混淆矩阵常见形式：

| 真实 / 预测 | 预测 0 | 预测 1 |
| --- | ---: | ---: |
| 真实 0 | TN | FP |
| 真实 1 | FN | TP |

## 9. KNN

KNN 的全称是 K-Nearest Neighbors，中文叫 K 近邻。

它的核心思想是：

> 看离新样本最近的 K 个邻居，大多数邻居是什么类别，新样本就预测成什么类别。

例如：

```text
K = 3
最近 3 个邻居中，有 2 个是类别 A，1 个是类别 B
那么新样本预测为类别 A
```

KNN 需要标准化，因为它依赖距离计算。

如果不同特征数值范围差异很大，距离会被数值大的特征主导。

K 值影响：

| K 值 | 影响 |
| --- | --- |
| K 太小 | 容易受噪声影响，可能过拟合 |
| K 太大 | 边界过于平滑，可能欠拟合 |

---

## 10. 朴素贝叶斯

朴素贝叶斯是一种基于概率的分类算法。

它的核心思想是：

> 根据已有信息不断更新某个类别的概率。

在文本分类中，模型会根据句子里出现的词判断类别。

例如：

```text
句子：我 喜欢 AI
```

看到“喜欢”以后，模型会认为：

```text
正面类别的概率更高。
```

`predict_proba` 输出两个数字时，通常表示：

```text
[属于类别 0 的概率, 属于类别 1 的概率]
```

例如：

```text
[0.38, 0.62]
```

说明：

```text
类别 0 概率 38%
类别 1 概率 62%
```

最终预测类别 1。

## 11. SVM

SVM 的全称是 Support Vector Machine，中文叫支持向量机。

它的核心思想是：

> 在不同类别之间找到一条最稳的分界线。

SVM 不只是找能分开的线，而是找：

```text
离两边样本都尽量远的分界线。
```

关键参数：

| 参数 | 含义 |
| --- | --- |
| `kernel` | 使用什么方式构造分类边界 |
| `C` | 对分类错误的容忍程度 |

常见 kernel：

| kernel | 说明 |
| --- | --- |
| linear | 线性边界 |
| rbf | 非线性边界，常用 |
| poly | 多项式边界 |
| sigmoid | 类似神经网络激活形式 |

在二维图中，可以通过背景色画出 SVM 的分类区域。

不同背景颜色交界处，就是 SVM 的分界线。

## 12. KMeans 聚类

KMeans 是无监督学习算法。

它不需要标签 `y`。

它的目标是：

> 把相似的数据分到同一组。

核心参数：

```python
n_clusters=3
```

表示希望模型把数据分成 3 组。

KMeans 会不断做两件事：

```text
1. 给每个样本分配最近的聚类中心
2. 根据分组结果重新计算聚类中心
```

直到结果稳定。

重要概念：

| 概念 | 含义 |
| --- | --- |
| cluster label | 聚类编号 |
| cluster center | 聚类中心 |
| inertia | 样本到所属中心的距离总和 |
| elbow method | 肘部法，用来选择 K |

肘部法的直觉是：

```text
K 增加时，inertia 会下降；
当前面下降很快、后面下降变慢时，拐点位置通常是较合适的 K。
```

## 13. PCA 降维

PCA 是 Principal Component Analysis，中文叫主成分分析。

它属于无监督学习。

PCA 的作用是：

> 用更少的新特征保留原始数据中的主要信息。

例如：

```text
原来有 4 个特征
通过 PCA 压缩成 2 个主成分
```

PCA 不需要 `y`，因为它不关心类别答案，只关心 `X` 的内部变化结构。

关键参数：

```python
n_components=2
```

表示保留 2 个主成分。

重要属性：

```python
explained_variance_ratio_
```

表示每个主成分保留了多少信息。

例如：

```text
[0.73, 0.23]
```

说明：

```text
第 1 个主成分保留 73% 信息
第 2 个主成分保留 23% 信息
两个加起来保留约 96% 信息
```

## 14. 集成学习：GBDT、XGBoost、LightGBM

集成学习的思想是：

> 多个弱模型组合起来，形成一个更强的模型。

本阶段主要学习了：

```text
GBDT
XGBoost
LightGBM
```

### 14.1 GBDT

GBDT 是梯度提升决策树。

它的思想是：

```text
第一棵树先预测
第二棵树修正第一棵树的错误
第三棵树继续修正前面的错误
不断迭代
```

### 14.2 XGBoost

XGBoost 可以理解为工程增强版 GBDT。

它常用于：

```text
结构化表格数据
机器学习比赛
工业建模
```

常见参数：

| 参数 | 含义 |
| --- | --- |
| `n_estimators` | 树的数量 |
| `learning_rate` | 学习率 |
| `max_depth` | 树最大深度 |
| `eval_metric` | 评价指标 |

### 14.3 LightGBM

LightGBM 也是梯度提升模型。

它的特点是：

```text
训练速度快
适合较大规模表格数据
常用于分类和回归任务
```

## 15. 异常检测

异常检测的目标是：

> 找出和大多数数据不太一样的样本。

本阶段使用的是：

```python
IsolationForest
```

中文叫孤立森林。

它的输出：

```text
1  = 正常
-1 = 异常
```

关键参数：

```python
contamination=0.1
```

表示预计大约 10% 的数据是异常。

`contamination` 越大，模型检测出的异常数量通常越多。

异常检测常见用途：

```text
异常交易检测
异常成绩检测
异常设备状态
异常医学样本
```

## 16. 时间序列机器学习

时间序列数据有时间顺序。

例如：

```text
第 1 天销量
第 2 天销量
第 3 天销量
...
```

时间序列预测不能随机打乱数据。

因为应该是：

```text
用过去预测未来
```

本阶段学习了通过 lag 特征把时间序列转换成机器学习问题。

例如：

| day | sales | lag_1 | lag_2 | lag_3 |
| ---: | ---: | ---: | ---: | ---: |
| 4 | 120 | 115 | 110 | 100 |

含义是：

```text
用前 1 天、前 2 天、前 3 天销量预测今天销量。
```

时间序列机器学习流程：

```text
构造 lag 特征
→ 删除缺失值
→ 按时间顺序划分训练集和测试集
→ 训练模型
→ 预测未来
```

在本阶段实验中，随机森林对持续上涨趋势的预测不理想。

原因是：

> 随机森林不擅长外推超过训练范围的趋势。

对于明显线性上涨的数据，线性回归可能更合适。

## 17. 交叉验证与参数搜索

单次训练集/测试集划分可能有偶然性。

交叉验证可以更稳定地评估模型。

### 17.1 交叉验证

例如 5 折交叉验证：

```text
把数据分成 5 份
每次拿 1 份做验证
剩下 4 份做训练
重复 5 次
最后取平均结果
```

常用代码：

```python
cross_val_score(model, X, y, cv=5, scoring="accuracy")
```

### 17.2 GridSearchCV

GridSearchCV 会尝试所有参数组合。

优点：

```text
搜索全面
```

缺点：

```text
速度慢
```

### 17.3 RandomizedSearchCV

RandomizedSearchCV 从参数空间里随机抽取部分组合。

优点：

```text
速度快
```

缺点：

```text
不一定找到最优组合
```

## 24. 阶段总结

第二阶段让我系统地完成了传统机器学习入门。

这一阶段最重要的是理解：

```text
数据 → 特征 X 和标签 y → 训练模型 → 预测 → 评估
```

同时也开始理解不同算法适合不同任务：

```text
回归模型适合预测连续值
分类模型适合预测类别
聚类模型适合自动分组
降维模型适合压缩和可视化数据
异常检测模型适合发现不正常样本
```

本阶段最重要的一句话是：

```text
机器学习不是只训练模型，而是要根据任务选择合适算法，并用评估指标判断模型是否真的有效。
```
